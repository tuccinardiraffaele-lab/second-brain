### Claude Code — Changelog recente

**v2.1.120 (aprile 25) — CRASH BUG, rollback automatico a v2.1.119**
- Bug critico: crash al resume di sessioni precedenti con `--resume` o `--continue`
- Auto-update ha riportato i client affetti a v2.1.119
- **Non aggiornare oltre 2.1.119 finché non esce il fix**

**v2.1.119 (aprile 23)**
- `/config` settings (theme, editor mode, verbose) ora persistono in `~/.claude/settings.json` con override per project/local/policy
- Nuova setting `prUrlTemplate` per puntare il badge PR footer a URL custom (es. GitLab, Bitbucket)
- `CLAUDE_CODE_HIDE_CWD`: nasconde working directory nel logo di startup
- `--from-pr` ora accetta GitLab MR, Bitbucket PR e GitHub Enterprise URL
- `--print` mode rispetta `tools:` e `disallowedTools:` dal frontmatter dell'agente

**Postmortem performance degradation** (conferma Anthropic, aprile 24)
Fortune e vari media riportano la conferma ufficiale: tre bug distinti tra marzo e aprile:
1. Riduzione intenzionale dello sforzo di ragionamento (per gestire latenza) — mal recepita
2. Bug di session-clearing — comportamento ripetitivo e perdita di memoria
3. Cambio system prompt per ridurre verbosità — ha danneggiato la qualità del coding

Tutti e tre corretti in v2.1.116 (aprile 20).
