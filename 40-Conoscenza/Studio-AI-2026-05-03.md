## Per fonte

### 1. Anthropic Blog & Research

**Claude Opus 4.7** — rilasciato il 16 aprile 2026, ora GA su tutti i prodotti Claude, API, Amazon Bedrock, Google Cloud Vertex AI, Microsoft Foundry.

Novità principali:
- **Visione**: acuity da 54,5% → 98,5%. Immagini fino a 2576px / 3,75MP (era 1568px / 1,15MP, +3×).
- **Effort `xhigh`**: nuovo livello tra `high` e `max` in Claude Code.
- **Coding**: SWE-bench Pro 64,3% (GPT-5.4 = 57,7%, Gemini 3.1 Pro = 54,2%). CursorBench: da 58% a 70%.
- **Prezzi**: invariati ($5/M input, $25/M output) ma il nuovo tokenizer produce fino al 35% in più di token per lo stesso testo → costo effettivo per chiamata potenzialmente superiore.

> [!warning] Impatto su [[Sistema-Agentico-Studio]]
> Se AIOS usa Opus 4.7 come modello base, tenere conto del tokenizer più costoso nelle stime di costo per chiamata. Il vantaggio visione è rilevante per agenti che processano documenti scansionati (fatture, buste paga, 730). L'effort `xhigh` può migliorare la qualità sulle task complesse di analisi contabile.

**Claude for Creative Work** — nuovi connettori MCP per professionisti creativi: Ableton (musica), Autodesk Fusion (3D modeling), Blender (3D), Resolume (visual arts live), SketchUp (3D), Splice (sample search). Non rilevante per il dominio consulenza/studio.

**Claude Security** (public beta dal 30 aprile 2026):
- Vulnerability scanning sul codebase con Opus 4.7.
- Disponibile per clienti Claude Enterprise; supporto Team e Max in arrivo.
- Spiega ragionamento, confidence, exploitability, efficacia della fix proposta.
- Funzionalità: scheduled scans, dismiss con motivazione documentata, export CSV/Markdown.
- Accessibile da claude.ai/security o dalla sidebar di claude.ai.
- "Hundreds of organizations" hanno già trovato issue che strumenti tradizionali non vedevano.

> [!tip] Potenziale uso su AIOS
> Quando AIOS sarà in produzione, Claude Security potrebbe essere utile per un security audit del codebase prima di espandere a clienti B2B.

### 3. Claude Code Changelog

**Versione 2.1.126** (più recente nel changelog):

Novità principali:
- `claude project purge [path]` — elimina tutto lo stato locale di un progetto (transcripts, tasks, file history, config entry). Supporta `--dry-run`, `-y`, `-i`, `--all`.
- `/model` picker ora elenca modelli dall'endpoint `/v1/models` del gateway quando `ANTHROPIC_BASE_URL` punta a un gateway Anthropic-compatible (utile per deployment enterprise custom).
- `claude auth login` ora accetta il codice OAuth incollato nel terminale quando il browser callback non raggiunge localhost — **fix esplicito per WSL2, SSH e container**.
- `--dangerously-skip-permissions` ora bypassa anche write su `.claude/`, `.git/`, `.vscode/`, shell config file.
- Auto mode: lo spinner diventa rosso quando un permission check si blocca.
- Fix **security**: `allowManagedDomainsOnly` / `allowManagedReadPathsOnly` venivano ignorati quando la sorgente di settings non aveva un blocco `sandbox`.
- Fix: incollare un'immagine >2000px non rompe più la sessione (downscale automatico).
- Windows: PowerShell 7 rilevato anche se installato via Microsoft Store, MSI senza PATH, o `.NET global tool`.
- Fix: caratteri giapponesi/coreani/cinesi non più corrotti su Windows in no-flicker mode.
- Fix memory: `unbounded memory growth` con molte immagini e `/usage` su grandi history.

**Versione 2.1.121** (precedente, novità notevoli):
- `alwaysLoad: true` su config MCP server — salta la tool-search deferral, tutti i tool sempre disponibili.
- `claude plugin prune` — rimuove dipendenze plugin orfane.
- Type-to-filter search box in `/skills`.
- PostToolUse hooks possono ora sostituire l'output del tool per **tutti** i tool (non solo MCP).

> [!warning] Impatto su [[Sistema-Agentico-Studio]] — WSL2 OAuth fix
> Il fix in 2.1.126 per `claude auth login` in WSL2 è direttamente rilevante: AIOS è sviluppato su `\\wsl.localhost\Ubuntu-22.04`. Se hai problemi di autenticazione OAuth durante lo sviluppo su WSL2, aggiorna Claude Code a 2.1.126+.

> [!warning] Impatto su [[Sistema-Agentico-Studio]] — `claude project purge`
> Utile per manutenzione del progetto AIOS se la history di sessioni diventa pesante. Da tenere a mente dopo il go-live del 10 maggio.
