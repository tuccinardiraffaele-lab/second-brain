#### Managed Agents (public beta — 8 aprile 2026)
Runtime hosted per agenti autonomi. Quattro blocchi fondamentali:
- **Agent** — definizione: modello, system prompt, tools, MCP server
- **Environment** — container cloud pre-configurato (Python/Node/Go, network egress default-deny)
- **Session** — istanza running con event log persistente (durable: sopravvive a disconnessioni)
- **Events** — stream SSE tra app e agente, tutti persistiti server-side

Pricing: tariffe token standard + **$0.08/ora di sessione** attiva (non si paga l'idle). Per task brevi (10 min) il surcharge è ~$0.013 — trascurabile rispetto ai token.

Stato: **public beta** — tutti gli account hanno accesso, nessuna whitelist. Sub-feature *Outcomes* e *Multiagent* in research preview (accesso separato).

## Action items

> [!warning] Impatto su [[Sistema-Agentico-Studio]]
> **Managed Agents è direttamente rilevante per AIOS.**
>
> Punti concreti da valutare:
> - **Nessun DevOps burden**: elimina la necessità di gestire container/infra — perfetto per uno studio piccolo senza team engineering
> - **Long-running tasks**: contabilizzazione, buste paga, F24 sono workflow multi-step da minuti/ore — esattamente il caso d'uso ottimizzato
> - **MCP + Vaults**: gestione OAuth per connettori esterni (GBSoftware-like) gestita da Anthropic, non da noi
> - **Durable sessions**: il padre carica documenti, l'agente processa in autonomia — nessun babysitting
> - **Rischio beta**: ancora public beta, comportamenti possono cambiare. Per il pilot (studio padre, scala ridotta) accettabile. Per B2B futuro, aspettare GA
>
> **Azione suggerita**: valutare se sostituire l'architettura loop custom di AIOS con Managed Agents nel refactor post-pilot. Non toccare ora (guardrail: nessun refactor fuori dalla critical path).

> [!info] Claude Code quality regression — impatto su sessioni di sviluppo
> Se noti output meno consistenti di Claude Code nelle sessioni di coding AIOS, è un problema noto e riconosciuto da Anthropic. Workaround attuale: usa Opus 4.7 invece di Sonnet 4.6 per task complessi se qualità degrada visibilmente.

> [!info] Claude Code sul Pro plan
> Da monitorare. Se venisse rimosso dal Pro, valutare upgrade a Max plan o Team. Impatto operativo basso nel breve, ma da tenere d'occhio nelle prossime settimane.
