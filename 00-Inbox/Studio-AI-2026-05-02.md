### 1. Anthropic Blog & Research

#### Claude Opus 4.7 — Generally Available
**Data**: fine aprile / inizio maggio 2026

- Nuovo flagship model, upgrade diretto su Opus 4.6
- Su 93-task coding benchmark: +13% task risolti, inclusi 4 task che né Opus 4.6 né Sonnet 4.6 risolvevano
- Migliore visione (immagini ad alta risoluzione) e output più curato per deliverable professionali
- **Self-verification su task lunghi** — il modello verifica il proprio output a fine task senza richiesta esplicita
- Pricing invariato: $5/MTok input, $25/MTok output; fino al 90% di risparmio con prompt caching

> [!warning] Impatto su [[Sistema-Agentico-Studio]]

### 3. Claude Code Changelog — v2.1.126 (1 maggio 2026)

Novità rilevanti:

| Tipo | Dettaglio |
|---|---|
| Feature | `/model` picker lista modelli da `/v1/models` del gateway quando `ANTHROPIC_BASE_URL` punta a un gateway compatibile |
| Feature | `claude project purge [path]` — cancella tutto lo stato Claude Code per un progetto (transcript, task, file history). Flags: `--dry-run`, `-y`, `-i`, `--all` |
| Fix critico | **LSP diagnostics pre-edit non appaiono più post-edit** — il modello non rilegge file appena modificati, eliminando un loop inefficiente in sessioni agentiche lunghe |
| Fix critico | **Retry su API overload ripristinato** — v2.1.110 aveva cappato i retry causando fallimenti invece di attesa; comportamento precedente ripristinato |
| Fix | CJK/multibyte text corruption (`U+FFFD`) in stream-json mode su chunk boundary UTF-8 |
| Fix | Bedrock: Sonnet 3.5 v2 ora usa il profilo di inferenza `us.` corretto |
| Fix | SDK/print mode preserva la risposta parziale dell'assistant in caso di interrupt |
| Fix | VSCode: warning banner se `settings.json` non è parsabile (le permission rule non venivano silenziosamente ignorate) |
| Windows | Git for Windows non più obbligatorio — PowerShell come fallback shell |
| CI/scripts | `claude ultrareview [target]` subcommand per uso non-interattivo |
| Skills | Variabile `${CLAUDE_EFFORT}` usabile nel contenuto delle skill; ricerca type-to-filter in `/skills` |

> [!tip] Rilevante per sviluppo quotidiano
> Il fix LSP e il ripristino del retry migliorano direttamente la stabilità delle sessioni di coding su [[Sistema-Agentico-Studio]] e [[Football-Trading]]. Aggiornare Claude Code.
