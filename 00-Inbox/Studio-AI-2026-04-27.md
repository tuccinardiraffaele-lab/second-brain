# Studio AI — 2026-04-27

## TL;DR

- **Claude Code quality crisis chiusa**: Anthropic ha ammesso 3 errori ingegneristici tra marzo e aprile (reasoning effort ridotto, bug reasoning history, cap 25 parole tra tool calls). Tutti risolti in v2.1.116 (20 aprile) dopo settimane di backlash utenti.
- **Opus 4.7 + Claude Design**: nuovo modello top (stesso pricing 4.6, nuovo tokenizer), affiancato da Claude Design — tool visuale per prototipi/slide/design, powered by Opus 4.7, preview per Pro/Max/Team/Enterprise.
- **Claude Code v2.1.117-119**: gli hooks possono ora chiamare MCP tools direttamente (`type: "mcp_tool"`); agent frontmatter `mcpServers` e `hooks` funzionano via `--agent`; effort default ripristinato a `high`.

- **Hooks possono invocare MCP tools direttamente** via `type: "mcp_tool"`

**v2.1.117**

- **Default effort ripristinato a `high`** per Pro/Max su Opus/Sonnet 4.6

- Patch di tutti e tre i bug del quality decline (reasoning effort, reasoning history, 25-word cap)

**v2.1.115**

- Effort level `xhigh` per Opus 4.7 (tra `high` e `max`)
- Auto mode disponibile per Max + Opus 4.7

### 4. Hacker News

**Claude Code quality post-mortem** (24 aprile)
Anthropic ha pubblicato un post di engineering in cui ammette i 3 errori che hanno degradato Claude Code tra marzo e aprile:
1. **4 marzo**: reasoning effort abbassato da `high` a `medium` per ridurre latenza → scelta sbagliata
2. **26 marzo**: bug che causava alla sessione di scartare il proprio reasoning history → il modello sembrava "dimentico" e consumava token in eccesso
3. **16 aprile**: system prompt cap a 25 parole tra tool calls → riduzione qualità coding misurata, riverted in 4 giorni

Tutti e tre risolti al 20 aprile (v2.1.116). Però il backlash ha portato utenti alla cancellazione degli abbonamenti.

**OpenClaw / harness di terze parti** (inizio aprile)
Anthropic ha prima vietato l'uso di harness terzi (es. OpenClaw) sulle subscription, poi ha fatto dietrofront e li ha riammessi.

**Claude Desktop App — native messaging bridge** (fine aprile)
Polemica su HN per un bridge di native messaging installato dall'app desktop di Claude senza disclosure esplicita. Thread attivo.

## Action items

> [!warning] Impatto su [[Sistema-Agentico-Studio]]
> **v2.1.118 — Hooks → MCP tools direttamente**: gli hooks possono ora invocare MCP tools senza passare per bash. Per AIOS questo apre un pattern architetturale pulito: evento (PostToolUse) → azione MCP (es. log su vault Obsidian, trigger agente successivo). Da valutare nell'integrazione vault AIOS.
>
> **v2.1.117-118 — Agent frontmatter mcpServers + hooks via --agent**: semplifica l'architettura multi-agente di AIOS. Si possono definire MCP server e hook direttamente nel frontmatter dell'agente, senza configurazione globale.
>
> **v2.1.117 — Effort default ripristinato a `high`**: se gli agenti AIOS usano l'effort di default, dopo marzo erano degradati a `medium`. Ora sono di nuovo a `high` — ricalibra le aspettative di qualità e latenza.

> [!warning] Impatto su [[Football-Trading]]
> **Nessun impatto diretto** nei contenuti di oggi. Il ripristino effort `high` (v2.1.117) vale anche per FT se usa agenti Claude Code, ma FT è in fase di accumulo dati autonoma — nessuna azione richiesta.

> [!tip] Da tenere d'occhio
> - **Opus 4.7 nuovo tokenizer**: quando AIOS userà Opus 4.7 (se/quando), i cost estimate per sessione cambieranno. Ricalcolare prima del go-live se si migra da 4.6.
> - **Claude Design**: potenzialmente utile post-go-live AIOS per produrre materiali presentazione B2C/B2B (slide, one-pager). Non prioritario ora.
> - **Mythos Preview + Project Glasswing**: interessante per il filone sicurezza, non urgente per i piloti attuali.
